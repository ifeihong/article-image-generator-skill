## 概要

**Article Image Generator** は、Codex、Trae、Claude Code 向けに設計された AI Skill です。文章コンテンツを視覚的に印象的なイラストに変換します。汎用 AI アートツールやストックフォト検索とは異なり、文章のナラティブ構造を理解し、アイデアの真の視覚的拡張となる画像を生成します —— 単なる装飾ではありません。

2 つの**キャラクターアンカーモード**、10 の**ビジュアルスタイル**、およびカスタマイズ可能な**アスペクト比**（16:9、9:16、4:3、3:4、1:1、21:9）と**枚数**（1-9 枚）を中心に構築されており、コンテンツのプラットフォーム、トーン、主題、読者の期待に合わせて創造的な出力を調整します。

> "イラストは説明するべきではない —— 拡張するべきだ。"

## 主な機能

### デュアルキャラクターアンカーモード

ブランド一貫性とコンテンツ固有の創造性の間で選択：

| 側面 | ブランドアンカーモード | コンテキストアンカーモード |
|------|---------------------|------------------------|
| **核心ロジック** | キャラクター参考画像をアップロード、すべてのイラストに同じ形象を使用 | 各文章のテーマに合わせた独自キャラクターを生成 |
| **最適な用途** | ブランドコンテンツ、シリーズ記事、個人 IP、企業ブログ | 深掘り記事、創造的実験、トレンド話題 |
| **キャラクター一貫性** | 最大（すべてのコンテンツで同じ形象） | 高（1 つの記事内で同じ形象） |
| **ブランド認知度** | 強（視覚的シグネチャ） | 微妙（コンテンツ優先アプローチ） |
| **創作自由度** | 既存キャラクターに制約 | 無制限、創造性を最大化 |

---

### カスタマイズ可能なアスペクト比と枚数

生成前に、Skill は 2 つの簡単な設定ステップを案内します：

**アスペクト比選択** —— ターゲットプラットフォームに合わせたキャンバス形状を選択：

| 比率 | 方向 | 最適な用途 |
|:---:|:---:|:---|
| **16:9** | 横長 | ブログ記事、YouTubeサムネイル、note、Zenn記事（デフォルト） |
| **9:16** | 縦長 | TikTok、Instagram Reels、YouTube Shorts、LINE VOOM、スマホ壁紙 |
| **4:3** | 横長 | PowerPoint、Keynote、講義スライド、レポート、プレゼンテーション |
| **3:4** | 縦長 | Instagramフィード、X/Twitter画像、Pinterest、Threads |
| **1:1** | 正方形 | Instagram、X/Twitter、LINEアイコン、商品写真、メルカリ |
| **21:9** | 超ワイド | Webサイトバナー、X/Twitterヘッダー、Pixiv、Twitch、ワイドスクリーン |

Skill はコンテンツタイプに基づいて賢く推奨しますが、任意の比率に上書きできます。

**枚数選択** —— 生成するイラストの枚数を選択（1-9 枚）：

| 文章の長さ | 推奨枚数 |
|:---|:---:|
| 短文（< 1000 語） | 1-3 枚 |
| 中篇（1000-3000 語） | 4-6 枚 |
| 長文（> 3000 語） | 7-9 枚 |

### コンセプトフォージ：4 つの創作方法論

**1. 直訳法** —— 文章の核心概念を直接視覚化
**2. 異質合成** —— 3 つの無関係な領域を強制的に組み合わせ
**3. 認知錯位** —— 尺度・機能・時空・感情の 4 つの錯位
**4. ランダムトリガー** —— 素材 + 実体 + 職業 + 感情の組み合わせ

---

### ユーザー確認フロー（コンテキストアンカーモード）

Skill はユーザーの承認なしにキャラクターを生成しません：
1. 文章テーマに基づいて 2-3 つのキャラクターコンセプトを提示
2. ユーザーの選択または修正提案を待つ
3. 承認されたキャラクター説明をロック
4. 記事のすべてのイラストに同一キャラクターを使用

---

### 多言語アノテーション対応

Skill は文章の言語を自動検出し、同じ言語でラベルを生成します。手動選択も可能：

| 検出言語 | ラベルスタイル | 例 |
|:---|:---|:---|
| **中文** | 横書き、2-8文字、手書き | 光芯片 / 数据流 / 瓶颈 |
| **English** | 横書き、1-3単語、手書き | Light Chip / Data Flow / Bottleneck |
| **日本語** | 横書きまたは縦書き、短いフレーズ | 光チップ / データ流 / ボトルネック |
| **韓国語** | 横書き、短いフレーズ | 광칩 / 데이터 흐름 / 병목 |

**自動検出**：文章テキストを分析（>50% 文字タイプ判定）。**手動上書き**：「日本語のラベルで」。

### クイックモード vs エキスパートモード

時間と制御ニーズに応じてワークフローを選択：

| 側面 | クイックモード | エキスパートモード |
|------|-------------|------------------|
| **ステップ** | 1ステップ自動検出 | 完全な9ステップ |
| **最適** | 高速出力、熟練ユーザー | 初回ユーザー、精密制御 |
| **パラメータ** | すべて自動検出 | ユーザーが逐次選択 |
| **戦略レポート** | スキップ | 生成前にオプション出力 |

**クイックモード例**：「この記事のイラストを作成」→ 日本語自動検出、16:9、6枚、ベーススケッチ。

**エキスパートモード例**：「エキスパートモード」→ モード → 言語 → アスペクト比 → 枚数 → スタイル →（オプション戦略レポート）→ 内容分析 → 生成。

---

### ハイブリッドスタイルミキシング（エキスパートモード）

2つのスタイルを組み合わせて独自の視覚効果を生み出す：

| プライマリ（60-70%） | セカンダリ（30-40%） | 結果 |
|:---|:---|:---|
| インクウォッシュ禅 | サイバーネオン | 「ネオン水墨」— 伝統的筆致に発光アクセント |
| クレイストップモーション | グリッチアート | 「破損した粘土」— 手作りフォルムにデジタルアーティファクト |
| レトロテック | オーガニックウォーム | 「バイオメカニカルビンテージ」— 古い機械に植物が生い茂る |

スタイル選択時に **[H] ハイブリッドモード** を選択してアクティベート。

### セッション履歴と再利用

生成後、パラメータを自動保存。簡単なコマンドで再利用：

| コマンド | アクション |
|:---|:---|
| 「前回と同じ設定」 | 最後のセッションを読み込み |
| 「デフォルト設定を使用」 | 保存済みデフォルトを読み込み |
| 「デフォルトとして保存」 | 現在のパラメータを保存 |
| 「履歴を表示」 | 最近10件を一覧 |
| 「セッション3を再利用」 | 指定セッションを読み込み |

履歴はローカル `assets/session-history.json` に保存 — データは外部に送信されません。

---

### シリーズキャラクター継続

シリーズ記事を書く？コンテキストアンカーキャラクターを保存して複数記事で再利用：

1. **保存**：「AI Weekly シリーズ用に保存」→ セッション履歴に保存
2. **再利用**：「AI Weekly のキャラクターを使用」→ 確認をスキップ、保存済みキャラクターを使用
3. **管理**：一覧、更新、退役、クローン

| 機能 | ブランドアンカー | シリーズキャラクター | コンテキストアンカー |
|------|-------------|------------------|------------------|
| 範囲 | すべてのコンテンツ | 1つのシリーズ | 1つの記事 |
| 永続性 | 永久 | 半永久 | 一時的 |
| 最適 | 個人IP | コラム、Newsletter | 単発記事 |

### イラスト戦略レポート（エキスパートモード）

生成前に戦略レポートをリクエストして、視覚的な思考を理解：

```
「先に戦略を見せて」
```

レポートには以下が含まれる：
- **内容分析**：記事タイプ、核心論点、ターゲット読者
- **視覚アンカーポイント**：どのセクションに画像が必要か、なぜか
- **認知負荷マップ**：テキストのみ vs 画像サポートセクション
- **キャラクター戦略**：なぜこのキャラクターが内容に適合するか
- **スタイル理由**：なぜこのスタイルがトーンに合うか
- **数量と配置**：画像数と配置ロジック

画像生成を教育可能なコンテンツデザインプロセスに変える。

### 初回使用ガイド

初回ユーザーは自動的に**ステップバイステップガイド**に入ります：

1. **ウェルカム**：フレンドリーな紹介、いつでもスキップ可能
2. **モード説明**：ブランドアンカー vs コンテキストアンカーの比較と用途
3. **言語選択**：各言語オプションの明確な説明
4. **アスペクト比ガイド**：ビジュアル形状プレビュー + プラットフォーム推奨
5. **枚数ガイド**：文字数→画像数マッピング + 認知的目的の説明
6. **スタイルギャラリー**：各スタイルに絵文字アイコン、説明、最適用途
7. **コンテンツ入力**：対応フォーマットの説明
8. **戦略レポート**：オプションの生成前分析
9. **キャラクター選択**：ガイド付きキャラクターコンセプト比較
10. **完了サマリー**：パラメータまとめ + 次回のヒント

スキップコマンド：「ガイドをスキップ」/「二度と表示しない」/「クイックモード」

---

### エラーハンドリング

一般的な問題に対して明確なガイダンスを提供：

| シナリオ | リカバリー |
|----------|-----------|
| 参照画像が見つからない | アップロードを促すかコンテキストアンカーに切替 |
| 参照画像が小さすぎる/ぼやけている | 要件を説明、代替案を提示 |
| 参照画像に複数のキャラクター | どれを使うか尋ね、クロップを提案 |
| 記事が短すぎる（<500字） | 1枚のヒーロー画像を提案 |
| 記事が長すぎる（>5000字） | セクション別生成を推奨 |
| キャラクター不一致 | キーワード強化、デザイン簡略化 |
| ラベル言語エラー | 明示的宣言で再生成 |
| スタイルが正しく適用されない | スタイル注入を強化して再生成 |
| 生成失敗（レート制限） | 30秒待ってリトライ、枚数を減らす |

すべてのエラーに：何が起きたか、なぜ、2-3つの実行可能なオプションを含む。

## クイックスタート

### 基本的な呼び出し

```
Article Image Generator を使用して、この記事のイラストを作成してください。
```

### モードの指定

```
ブランドアンカーモードを使用。    ← assets/ip-reference/ の参考画像を使用
コンテキストアンカーモードを使用。  ← カスタムキャラクターを生成
```

### アスペクト比と枚数の指定

```
9:16 アスペクト比、6 枚を使用。           ← 比率 + 枚数を直接指定
Article Image Generator を使用、4:3、3 枚。  ← 比率 + 枚数
```

### スタイルの指定

```
サイバーネオンスタイルを使用。
```
