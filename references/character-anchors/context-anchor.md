# 自由角色模式 — 角色生成规范

## 概述
自由角色模式不为配图绑定固定IP，而是根据每篇文章的主题、情绪、核心概念，为每张图（或每组图）定制专属角色。适合追求内容原生感、脑洞最大化、视觉丰富度的场景。

## 核心原则
- **一主题一角色**：同一篇文章的所有配图必须使用同一个角色，角色形象保持一致
- **角色即隐喻**：角色本身应该是文章概念的视觉化延伸
- **怪诞优先**：鼓励跨界混搭、尺度错位、功能错位
- **动作即表达**：角色必须承担核心动作，不能只是装饰
- **用户确认**：生成前必须向用户展示角色方案并获得确认

## 角色生成方法

### 方法一：主题直译法
把文章核心概念直接翻译成视觉角色。

| 文章主题 | 角色设计 | 逻辑 |
|---------|---------|------|
| 光芯片瓶颈 | 一只被挤扁的黑色小怪物 | 瓶颈=挤压 |
| 数据洪流 | 一条由0和1组成的鱼 | 数据=水流 |
| 供需缺口 | 一个拿着卷尺的裂缝测量员 | 缺口=需要测量 |

### 方法二：跨界混搭法
从 `references/metaphor-engine/cross-domain-matrix.md` 选取不相关的领域组合。

步骤：
1. 确定文章核心概念
2. 从矩阵中选一个"结构来源"、一个"质感来源"、一个"动作来源"
3. 强制组合成一个角色

示例：
- 结构来源：邮局 | 质感来源：生物 | 动作来源：攀岩
- 角色：一个由信封和邮票拼成的软体生物，正在岩壁上攀爬

### 方法三：荒诞逻辑法
使用 `references/metaphor-engine/absurd-logic.md` 的四种错位：

1. **尺度错位**
   - 微观放大：一只巨大的细菌在操作光刻机
   - 宏观缩小：一座城市缩成一个芯片大小

2. **功能错位**
   - 用A做B的事：用咖啡杯装闪电（能量存储）
   - 让B做A的事：让数据流去耕田（数据处理）

3. **时空错位**
   - 古代做现代：用算盘处理AI数据
   - 未来做古代：用激光笔写毛笔字

4. **情感错位**
   - 给机器情绪：焦虑的芯片、懒惰的光纤
   - 给抽象概念身体：供需缺口像伤口、增长曲线像藤蔓

### 方法四：随机触发法
当没有明确方向时，用以下随机组合生成角色：

```
角色 = [材质] + [生物/非生物] + [职业/功能] + [情绪]

材质池：金属、玻璃、纸、木头、布料、光、影子、液体
生物/非生物：猫、鸟、鱼、树、石头、云朵、齿轮、灯泡
职业/功能：邮差、守门员、厨师、侦探、工人、魔术师
情绪：焦虑、冷静、兴奋、困惑、认真、慵懒

示例：
- 纸 + 鸟 + 邮差 + 焦虑 = 一只由皱巴巴的纸折成的鸟，背着邮包，表情焦虑
- 光 + 树 + 守门员 + 冷静 = 一棵由光线编织的树，站在门前，表情冷静
```

## 角色类型库

### 按内容领域推荐

**科技/AI主题**
- 数据精灵：由光点、代码、线路组成的半透明生物
- 机械生物：有机+机械混搭，如齿轮心脏的猫
- 代码幽灵：漂浮的字符聚合体，有模糊的人形

**商业/金融主题**
- 数字商人：由图表、数字、货币符号拼成的人形
- 西装怪兽：穿西装但头部是非人生物（如灯泡、计算器）
- 趋势藤蔓：由上升曲线构成的植物状生物

**抽象/方法论主题**
- 概念容器：身体是空箱子/瓶子/口袋，里面装着抽象物
- 关系网：由线条和节点构成的人形，线条连接着不同物体
- 状态变形者：身体可以变形以表达不同状态（膨胀=增长，收缩=衰退）

**自然/环保主题**
- 元素精灵：火、水、土、风的拟人化
- 共生体：两种不同生物的融合体
- 时间植物：生长速度极快的植物，叶片上显示时间

## 角色设计检查清单

生成角色后检查：
- [ ] 角色是否与文章主题有关联？（即使是荒诞的关联）
- [ ] 角色是否能在画面中承担核心动作？
- [ ] 角色是否有至少一个记忆点特征？
- [ ] 角色是否避免了"可爱吉祥物"陷阱？
- [ ] 如果去掉角色，画面的核心意思是否还能成立？（如果不能，说明角色是核心）

## 与固定IP模式的对比

| 维度 | 固定IP模式 | 自由角色模式 |
|------|-----------|------------|
| 角色一致性 | 高（同角色反复出现） | 高（同一文章内同一角色） |
| 品牌感 | 强 | 弱 |
| 内容贴合度 | 中（角色需适配各种主题） | 高（角色为主题定制） |
| 创作自由度 | 低（受角色限制） | 高（无限制） |
| 系列感 | 强 | 弱 |
| 脑洞空间 | 小 | 大 |
| 适用场景 | 品牌内容、系列文章、个人IP | 单篇文章、创意内容、实验性项目 |

## 用户确认流程（必须执行）

### 步骤1：提供角色方案
根据文章主题，生成 2-3 个角色方案，每个方案包含：
- **角色名称**
- **外形描述**（材质、形状、标志性特征）
- **性格特点**
- **与文章的关联逻辑**
- **生成方法**（主题直译/跨界混搭/荒诞逻辑/随机触发）

### 步骤2：等待用户选择
- 用户选择其中一个方案
- 或用户提出修改意见
- 或用户提供自己的角色想法

### 步骤3：固化角色描述
用户确认后，将角色描述固化为标准格式，用于该文章所有配图：

```markdown
## 本文专属角色：[角色名称]

### 基础信息
- 名称：
- 材质/构成：
- 主色调：
- 身高比例：

### 外形特征
- 头部：
- 身体：
- 四肢：
- 标志性配饰（必须出现）：

### 表情风格
- 默认表情：
- 认真时：
- 困惑时：
- 兴奋时：

### 绝对不变项
生成时必须检查，以下元素不能改变或省略：
- [ ] 特征A
- [ ] 特征B
- [ ] 特征C
```

### 步骤4：统一使用
该文章的所有配图必须使用上述固化角色描述，确保形象一致。

## 使用建议
- 系列文章（3篇以上同一主题）→ 可考虑固定一个临时角色，但每篇文章仍需用户确认
- 单篇文章 → 一个角色贯穿所有配图
- 用户说"要脑洞""要好玩""要不一样" → 优先用荒诞逻辑法和跨界混搭法
