# Article Image Generator

> 一个为中文文章智能生成 16:9 横版配图的 Codex/Trae Skill。支持固定 IP 模式（品牌角色一致性）和自由角色模式（内容原生定制），内置 10 种可选视觉风格。

---

## 核心特色

### 双模式 IP 系统
- **固定 IP 模式**：上传角色三视图，所有配图使用同一角色，适合品牌内容、系列文章、个人 IP
- **自由角色模式**：根据文章主题定制专属角色，同一篇文章的所有配图保持同一角色形象，适合单篇深度内容、创意实验

### 10 种可选视觉风格
生成前提示用户选择，覆盖从手绘到赛博霓虹的完整风格谱系：基础手绘、复古科技、有机温暖、极简几何、混合媒介、水墨意境、赛博霓虹、木刻版画、黏土定格、故障艺术。

### 智能角色设计引擎
- **主题直译法**：将文章核心概念直接翻译为视觉角色
- **跨界混搭法**：从结构来源 × 质感来源 × 动作来源三维矩阵中强制组合
- **荒诞逻辑法**：尺度错位、功能错位、时空错位、情感错位四种手段
- **随机触发法**：材质 + 生物/非生物 + 职业/功能 + 情绪的组合池

### 用户确认流程（自由模式）
生成前向用户展示 2-3 个角色方案，等待用户选择后再固化角色描述，确保形象符合预期。

### 专业工作流
从模式确认 → 风格选择 → 正文消化 → 配图策略（shot list）→ 单张生成 → 质量检查 → 保存交付，全流程标准化。

## 安装方法

### 方式一：直接克隆（推荐）

```bash
git clone https://github.com/your-username/article-image-generator-skill.git
```

## 使用方法

### 基本使用

在 Codex/Trae 中直接调用：

```
使用 Article Image Generator 为这篇文章生成配图
```

或提供文章正文：

```
请根据以下内容生成配图：
[粘贴文章正文]
```

### 完整工作流

Skill 会自动执行以下步骤：

1. **模式确认**：检查是否有固定 IP 参考图，询问用户偏好
2. **风格选择**：展示 10 种风格菜单，等待用户选择
3. **正文消化**：分析文章核心观点、认知转折、配图需求
4. **配图策略**：输出 shot list（每张图的位置、主题、结构类型）
5. **角色确认**（自由模式）：提供 2-3 个角色方案，等待用户选择
6. **单张生成**：按顺序生成每张配图
7. **质量检查**：对照 QA 清单检查并迭代
8. **保存交付**：图片保存到 `assets/<article-slug>-illustrations/`

### 模式切换指令

```
切换到固定IP模式    → 使用 assets/ip-reference/ 下的参考图
切换到自由角色模式  → 为每篇文章定制新角色
这次用固定IP        → 单次任务使用固定IP
```

### 风格选择

生成前会自动提示：

```
请选择配图风格（输入数字或风格名）：
【01】基础手绘 — 纯白背景，黑色手绘线稿，怪诞清爽（默认）
【02】复古科技 — 泛黄工程手册，机械制图感
【03】有机温暖 — 植物脉络，水彩晕染，自然生长
【04】极简几何 — 包豪斯网格，精确分割，克制色彩
【05】混合媒介 — 照片+手绘，真实物体混搭
【06】水墨意境 — 东方水墨，留白意境，笔触浓淡
【07】赛博霓虹 — 霓虹光效，暗色背景，科技感
【08】木刻版画 — 强烈黑白，刀刻纹理，民间艺术
【09】黏土定格 — 3D黏土，手工捏制，圆润可爱
【10】故障艺术 — 数字故障，条纹错位，数据损坏感
```

### 固定 IP 模式：上传角色参考图

1. 准备角色三视图（正面 + 侧面 + 背面，或完整形象图）
2. 放入 `assets/ip-reference/character-3view.png`
3. 调用 skill 时选择固定 IP 模式
4. 所有配图将自动使用该角色

### 自由角色模式：定制角色

1. 调用 skill 时选择自由角色模式
2. Skill 会根据文章主题生成 2-3 个角色方案
3. 用户选择或修改方案
4. 角色描述固化后，所有配图使用同一角色

## 技术说明

### 关于参考图上传（固定IP模式）

当前 Codex/Trae 环境的 `GenerateImage` 工具**暂不支持 `reference_image` 参数直接上传参考图**。固定 IP 模式目前通过以下方式实现角色一致性：

1. **文字描述强化**：在提示词中嵌入完整的角色外貌描述（颜色、形状、比例、标志性特征）
2. **GPT Image 2 的上下文一致性**：同一会话内多轮生成，模型会参考历史输出维持角色形象
3. **固化角色描述**：生成前将角色描述固化为标准格式，所有配图复用同一描述

**未来改进**：当底层工具支持参考图参数后，可在 `fixed-ip.md` 中启用 `reference_image` 和 `reference_strength` 参数，实现真正的图像级角色锁定。

### 关于 GPT Image 2 的多图一致性

GPT Image 2（ChatGPT Images 2.0）支持以下特性：
- 单条提示词可生成 8 张保持角色和风格一致的图片
- 多轮对话式改图：同一会话内引用历史输出，维持角色形象
- 4K 分辨率、精准文字渲染

本 skill 利用同一会话上下文机制，在自由角色模式下通过固化角色描述 + 会话上下文引用，最大化角色一致性概率。

## 注意事项

1. **角色一致性**：同一篇文章的所有配图必须使用同一个角色，这是硬性规则
2. **用户确认**：自由模式下，未经用户确认不得直接生成角色
3. **风格注入**：选定风格后，所有提示词必须注入对应风格声明
4. **QA 检查**：生成后必须对照 `qa-checklist.md` 检查
5. **避免复刻**：每次为文章生成新隐喻，不照搬旧案例构图
