## 项目简介

**Article Image Generator** 是一款专为中文内容创作者设计的 AI 配图生成 Skill，基于 Codex / Trae / Claude Code 等 AI 编程助手运行。它能将文章中的核心观点、数据逻辑、概念隐喻转化为**清爽、怪诞、有创意**的 16:9 横版插图，彻底解决"写文章容易，找配图难"的痛点。

不同于传统的图库搜索或通用 AI 绘画工具，本 Skill 深度理解中文文章的叙事结构，通过**双模式 IP 系统**和**10 种视觉风格引擎**，为每篇文章量身定制视觉语言 —— 从科技行业分析到人文哲思，从商业数据报告到创意故事，都能找到最契合的视觉表达。

> "让文章配图不再是点缀，而是内容的延伸。"

## 核心功能

### 双模式 IP 系统：品牌一致性 vs 内容原生感

| 维度 | 固定 IP 模式 | 自由角色模式 |
|------|-------------|-------------|
| **核心逻辑** | 上传角色三视图，所有配图使用同一角色 | 根据文章主题定制专属角色 |
| **适用场景** | 品牌内容、系列文章、个人 IP、企业号 | 单篇深度内容、创意实验、热点追踪 |
| **角色一致性** | 极高（同一角色反复出现） | 高（同一文章内同一角色） |
| **品牌感** | 强（形成视觉记忆点） | 弱（角色为内容服务） |
| **创作自由度** | 受角色限制 | 无限制，脑洞最大化 |
| **用户操作** | 上传参考图 → 自动复用 | 确认角色方案 → 开始生成 |

**固定 IP 模式**适合需要长期运营的内容品牌。例如：科技博主上传一个"西装猴子"三视图，所有关于芯片、AI、互联网的文章都出现这只猴子，读者看到猴子就知道是你的文章。

**自由角色模式**适合追求内容原生感的场景。例如：写一篇关于"光芯片产业"的文章，Skill 会生成"光之搬运工"（一个由光线编织的人形角色）作为专属视觉符号，这个角色只在这篇文章中出现，与内容完美契合。

### 智能角色设计引擎：四种创意方法论

自由角色模式内置四套角色生成方法论，让配图角色不再是随机产物，而是文章概念的视觉化延伸：

**1. 主题直译法**
将文章核心概念直接翻译为视觉角色。例如：
- "光芯片瓶颈" → 一只被挤扁的黑色小怪物（瓶颈=挤压）
- "数据洪流" → 一条由 0 和 1 组成的鱼（数据=水流）

**2. 跨界混搭法**
从三个不相关领域各取一个元素强制组合：
- 结构来源（厨房/邮局/工厂/森林）× 质感来源（电路板/纸张/玻璃/植物）× 动作来源（攀岩/烹饪/侦探/航海）
- 示例：邮局（结构）+ 纸张（质感）+ 烹饪（动作）= 一个由信件和信封构成的厨房，角色正在"烹饪"信息

**3. 荒诞逻辑法**
通过四种错位打破常规思维：
- **尺度错位**：微观放大（芯片放大成城市）或宏观缩小（海洋缩小成一杯水）
- **功能错位**：用雨伞当漏斗、让大象穿针
- **时空错位**：用算盘处理 AI 数据、用激光笔写毛笔字
- **情感错位**：焦虑的芯片、懒惰的光纤、给抽象概念赋予身体

**4. 随机触发法**
当没有明确方向时，从组合池中随机抽取：
- 材质（金属/玻璃/纸/木头/光）+ 生物（猫/鸟/鱼/树/齿轮）+ 职业（邮差/厨师/侦探）+ 情绪（焦虑/冷静/兴奋）

### 用户确认流程（自由模式）

未经用户确认，Skill 不会直接生成角色。流程如下：

1. **提供方案**：根据文章主题生成 2-3 个角色方案，每个包含角色名称、外形描述、性格特点、与文章的关联逻辑
2. **等待选择**：用户选择方案、提出修改意见、或提供自己的想法
3. **固化描述**：将确认后的角色描述固化为标准格式
4. **统一使用**：该文章所有配图使用同一角色描述，确保形象一致

### 专业工作流：8 步标准化配图

```
模式确认 → 风格选择 → 正文消化 → 配图策略（Shot List）
    ↓
角色确认（自由模式）→ 单张生成 → 质量检查 → 保存交付
```

**Step 1: 模式确认**
检查 `assets/ip-reference/` 是否有参考图，询问用户偏好，确认后读取对应模式文档。

**Step 2: 风格选择**
展示 10 种风格菜单，用户输入数字或风格名即可选择。

**Step 3: 正文消化**
分析文章核心观点、认知转折点、哪些内容适合用图解释、哪些只适合文字。

**Step 4: 配图策略（Shot List）**
输出每张图的详细规划：放置位置、主题、核心意思、结构类型、角色动作、建议元素、中文标注词。默认 4-8 张。

**Step 5: 角色确认（自由模式）**
提供 2-3 个角色方案，等待用户选择。

**Step 6: 单张生成**
按顺序生成每张配图，提示词包含：16:9 横版声明、风格注入、角色描述、构图指令、约束条件。

**Step 7: 质量检查**
对照 QA 清单检查：角色是否承担核心动作、画面是否太满、是否像 PPT、中文标注是否可读、风格是否统一。

**Step 8: 保存交付**
图片保存到 `assets/<article-slug>-illustrations/`，按顺序命名 `01-topic-name.png`。

## 完整工作流详解

### 1. 模式确认

Skill 首先判断使用哪种模式：

| 触发条件 | 使用模式 |
|---------|---------|
| 用户明确说"用我的 IP""用固定角色""参考我的角色" | 固定 IP 模式 |
| `assets/ip-reference/` 目录下存在角色参考图 | 默认使用固定 IP 模式 |
| 用户说"自由角色""每图不同""脑洞大""不绑定角色" | 自由角色模式 |
| 无参考图且用户未指定 | 自由角色模式（默认） |

**模式切换指令**：
- "切换到固定 IP 模式" → 检查参考图是否存在，有则切换，无则提示上传
- "切换到自由角色模式" → 直接切换
- "这次用固定 IP，下次用自由角色" → 按单次任务执行

### 2. 风格选择

Skill 会自动展示风格菜单：

```
请选择配图风格（输入数字或风格名）：

【0】基础手绘 — 纯白背景，黑色手绘线稿，怪诞清爽（默认）
【1】复古科技 — 泛黄工程手册，机械制图感
【2】有机温暖 — 植物脉络，水彩晕染，自然生长
【3】极简几何 — 包豪斯网格，精确分割，克制色彩
【4】混合媒介 — 照片+手绘，真实物体混搭
【5】水墨意境 — 东方水墨，留白意境，笔触浓淡
【6】赛博霓虹 — 霓虹光效，暗色背景，科技感
【7】木刻版画 — 强烈黑白，刀刻纹理，民间艺术
【8】黏土定格 — 3D 黏土，手工捏制，圆润可爱
【9】故障艺术 — 数字故障，条纹错位，数据损坏感

直接回复数字（如"3"）或风格名即可。
```

### 3. 正文消化

Skill 读取文章后，会提炼以下信息：
- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

**配图原则**：不要平均配图。优先选择"认知锚点"：核心判断、断点、闭环、分流、前后对比、承接路径、角色状态变化。

### 4. 配图策略（Shot List）

Skill 输出每张图的详细规划：

```
配图策略（共 6 张）：

01 - 瓶颈挤压
位置：第 1 段后
主题：光芯片产能瓶颈
结构：概念隐喻
角色：光之搬运工（一个由光线编织的人形，背着光芯片）

标注：产能缺口 / 挤压感 / 2025

02 - 供需裂缝
位置：第 2 段后
...
```

### 5. 角色确认（自由模式）

在输出 Shot List 之前，Skill 会先提供角色方案：

```
根据文章主题，我设计了 3 个角色方案：

【方案 A】光之搬运工
- 外形：由光线编织的人形，半透明，身体有光纤纹理
- 性格：认真、沉默、略带疲惫
- 关联：直接对应"光芯片"主题
- 生成方法：主题直译法

【方案 B】硅基园丁
- 外形：半人半植物，身体是硅晶圆，头发是电路板
...

请选择一个方案，或提出修改意见。
```

### 6. 单张生成

Skill 按顺序生成每张图，提示词包含：
- 16:9 横版中文正文配图声明
- 风格声明（来自 style-variants.md）
- 角色描述（固定 IP 或自由角色）
- 构图指令（结构类型、动作、元素）
- 约束条件（禁止 PPT、禁止可爱、留白比例等）

### 7. 质量检查

生成后对照 QA 清单检查：
- [ ] 角色是否承担核心动作（不只是装饰）
- [ ] 画面是否太满（主体不超过 60%）
- [ ] 是否像 PPT/流程图/课程课件
- [ ] 中文标注是否少、短、能读
- [ ] 颜色使用是否正确（橙=主路径、红=重点、蓝=补充）
- [ ] 固定 IP 模式：角色是否与参考图一致

### 8. 保存交付

02-supply-demand-gap.jpg — 供需裂缝，最稳
03-monopoly-wall.jpg — 垄断高墙，可选
04-substitution-ladder.jpg — 替代路线，最稳

## 风格系统

### 基础风格 DNA

所有风格的共同基础：
- **16:9 横版**中文正文配图
- **纯白背景**：不要米色、暖灰、纸张纹理、渐变、阴影
- **黑色手绘线稿**：细线、轻微抖动、不机械、不矢量
- **大量留白**：主体占画面 40%-60%，至少 35% 空白
- **少量中文手写批注**：最多 5-8 处，每处 2-8 个字
- **一张图只讲一个核心动作、结构、状态或隐喻**

### 颜色规范

| 颜色 | 用途 |
|------|------|
| 黑色 | 主体线稿、角色、框线、结构、主要文字 |
| 红色 | 重点批注、问题、情绪点、关键提醒、结果 |
| 橙色 | 主流程、路径、箭头、从 A 到 B 的移动关系 |
| 蓝色 | 补充说明、系统状态、第二层解释、AI/自动化提示 |

### 风格禁忌

- 不要商业插画、PPT 信息图、正式流程图
- 不要课程课件、可爱卡通海报、儿童插画
- 不要复杂架构图、精致扁平插画、科技感 UI
- 不要真实 App 截图、复杂背景、渐变、阴影
- 不要把每个节点都解释清楚
- 不要左上角写"Workflow / 系统架构图 / 常见坑"等类型标题

## 技术原理

### 关于参考图上传（固定 IP 模式）

当前 Codex/Trae 环境的 `GenerateImage` 工具**暂不支持 `reference_image` 参数直接上传参考图**。固定 IP 模式通过以下方式实现角色一致性：

1. **文字描述强化**：在提示词中嵌入完整的角色外貌描述（颜色、形状、比例、标志性特征）
2. **GPT Image 2 的上下文一致性**：同一会话内多轮生成，模型参考历史输出维持角色形象
3. **固化角色描述**：生成前将角色描述固化为标准格式，所有配图复用同一描述

**未来改进**：当底层工具支持参考图参数后，可在 `fixed-ip.md` 中启用 `reference_image` 和 `reference_strength` 参数，实现真正的图像级角色锁定。

### 关于 GPT Image 2 的多图一致性

GPT Image 2（ChatGPT Images 2.0）支持：
- 单条提示词生成 8 张保持角色和风格一致的图片
- 多轮对话式改图：同一会话内引用历史输出，维持角色形象
- 4K 分辨率、精准文字渲染

本 Skill 利用同一会话上下文机制，在自由角色模式下通过固化角色描述 + 会话上下文引用，最大化角色一致性概率。

## 常见问题

### Q1: 固定 IP 模式生成的角色与参考图不一致怎么办？

A: 当前环境限制下，可通过以下方式改善：
1. 在提示词中强化角色描述的关键词
2. 确保参考图清晰、完整、白底
3. 简化角色设计（减少配件和颜色数量）
4. 未来底层工具支持 reference_image 后将彻底解决

### Q2: 自由模式生成的角色可以保存下来下次用吗？

A: 可以。将角色描述保存到 `assets/ip-reference/character-3view.png`（文字描述截图），下次即可作为固定 IP 使用。

### Q3: 生成的图片可以商用吗？

A: 取决于底层图像生成模型的使用条款。GPT Image 2 生成的图片通常允许商用，但建议查阅 OpenAI 最新使用政策。

### Q4: 支持英文文章吗？

A: 当前版本主要针对中文文章优化（中文标注、排版习惯）。英文文章可以使用，但标注和排版可能不够自然。

### Q5: 生成一张图需要多久？

A: 取决于底层模型和分辨率。GPT Image 2 通常 5-15 秒/张，4K 分辨率可能需要 20-30 秒。
