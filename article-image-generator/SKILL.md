# Article Image Generator — 文章配图生成器

## 核心定位
为中文文章设计和生成 16:9 横版正文配图。把文章里的关键判断、流程、结构、状态或隐喻，变成一张清爽、怪诞、有创意、可读但不说明书的手绘解释图。

支持两种角色模式，用户可自由选择：
- **固定IP模式**：使用用户提供的角色三视图，每张图出现同一个角色，适合品牌统一性
- **自由角色模式**：每篇文章/主题定制专属角色，**同一篇文章的所有配图使用同一个角色**，适合内容原生感和脑洞最大化

## 双模式系统

### 模式选择规则

| 触发条件 | 使用模式 |
|---------|---------|
| 用户明确说"用我的IP""用固定角色""参考我的角色" | 固定IP模式 |
| `assets/ip-reference/` 目录下存在角色参考图 | 默认使用固定IP模式 |
| 用户说"自由角色""每图不同""脑洞大""不绑定角色" | 自由角色模式 |
| 无参考图且用户未指定 | 自由角色模式（默认） |

### 模式切换指令
- 用户说"切换到固定IP模式" → 检查 `assets/ip-reference/` 是否有参考图，有则切换，无则提示上传
- 用户说"切换到自由角色模式" → 直接切换
- 用户说"这次用固定IP，下次用自由角色" → 按单次任务执行，不修改默认设置

## 工作流

### 1. 模式确认
首先判断使用哪种模式：
- 检查 `assets/ip-reference/` 是否有图片文件
- 询问用户偏好（如未明确）
- 确认后读取对应模式参考文档

### 2. 风格选择（新增）
**在消化正文之前，必须先让用户选择画风。**

向用户展示风格菜单：
```
请选择配图风格（输入数字或风格名）：

【0】基础手绘 — 纯白背景，黑色手绘线稿，怪诞清爽（默认）
【1】复古科技 — 泛黄工程手册，机械制图感
【2】有机温暖 — 植物脉络，水彩晕染，自然生长
【3】极简几何 — 包豪斯网格，精确分割，克制色彩
【4】混合媒介 — 照片+手绘，真实物体混搭
【5】水墨意境 — 东方水墨，留白意境，笔触浓淡
【6】赛博霓虹 — 霓虹光效，暗色背景，科技感
【7】木刻版画 — 强烈黑白，刀刻纹理，民间艺术
【8】黏土定格 — 3D黏土，手工捏制，圆润可爱
【9】故障艺术 — 数字故障，条纹错位，数据损坏感

直接回复数字（如"3"）或风格名即可。
```

- 用户选择后，确认选择
- 在后续所有提示词中注入对应风格声明
- 用户说"默认""基础""不选" → 使用 0. 基础手绘

### 3. 消化正文
读取用户给的正文、链接、Markdown、Notion 或截图。提炼：
- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择"认知锚点"：核心判断、断点、闭环、分流、前后对比、承接路径、角色状态变化。

### 4. 先出配图策略
如果用户说"分析怎么配图"，先给 shot list。每张图写清楚：
- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 角色在图里做什么（固定IP模式）/ 角色设计方向（自由角色模式）
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要超过 9 张。

**自由角色模式额外步骤：**
在输出 shot list 之前，**必须先向用户确认角色设计方向**：
1. 根据文章主题，提供 2-3 个角色方案（每个方案包含：角色名称、外形描述、性格特点、与文章的关联逻辑）
2. 说明每个方案的生成方法（主题直译/跨界混搭/荒诞逻辑/随机触发）
3. 等待用户选择或提出修改意见
4. 用户确认后，再输出 shot list 并开始生成

**禁止**：未经用户确认直接生成角色并出图

### 5. 单张生成
如果用户明确要求"生成"，不要停下来等确认；每张单独生成。

**固定IP模式生图参数：**
```
- 提示词：必须包含完整的角色描述（来自 fixed-ip.md）
- 参考图：传入 assets/ip-reference/ 下的角色图
- 参考强度：0.6-0.8（高一致性，保留创作空间）
- 角色必须在每张图中承担核心动作
- 风格声明：注入用户选择的风格（见 style-variants.md）
```

**自由角色模式生图参数：**
```
- 提示词：包含为该主题定制的角色描述（同一篇文章的所有图使用同一角色描述）
- 无参考图：纯文本驱动
- 同一篇文章的所有配图必须使用同一个角色，角色形象保持一致
- 每张图的角色动作和姿态可以不同，但核心外形、材质、标志性特征必须一致
- 鼓励跨界混搭和荒诞隐喻
- 风格声明：注入用户选择的风格（见 style-variants.md）
```

每张图提示词必须包含：
- 16:9 横版中文正文配图
- 风格声明（来自 style-variants.md）
- 黑色手绘线稿（基础风格）或对应风格的线条定义
- 少量红色/橙色/蓝色中文手写批注（根据风格调整）
- 大量留白（根据风格调整比例）
- 角色作为核心动作主体
- 禁止 PPT、商业插画、幼稚可爱、复杂架构、左上角类型标题

### 5. 检查与迭代
生成后检查 `references/qa-checklist.md`。常见问题：
- 角色只是装饰（未承担核心动作）
- 画面太满
- 太像流程图/PPT
- 中文太多或错字严重
- 左上角出现类型标题
- 画风太可爱、幼稚、死板
- 背景不是干净白底
- **固定IP模式额外检查**：角色形象是否与参考图一致？标志性特征是否出现？

### 6. 保存交付
最终图保存到：
```text
assets/<article-slug>-illustrations/
```
按顺序命名：
```text
01-topic-name.png
02-topic-name.png
```

## 输出口径
生成前：shot list 短而准。
生成后：报告生成了几张、每张用途、保存路径、哪些图最稳/可选。
不要长篇解释风格理论；让图自己说话。
