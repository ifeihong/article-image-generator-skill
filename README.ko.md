## 개요

**Article Image Generator**는 Codex, Trae, Claude Code용으로 설계된 AI Skill입니다. 텍스트 콘텐츠를 시각적으로 인상적인 일러스트로 변환합니다. 범용 AI 아트 도구나 스톡 사진 검색과 달리, 글의 서사 구조를 이해하고 아이디어의 진정한 시각적 확장이 되는 이미지를 생성합니다 —— 단순한 장식이 아닙니다.

2가지 **캐릭터 앵커 모드**, 10가지 **비주얼 스타일**, 그리고 사용자 정의 가능한 **화면비**（16:9, 9:16, 4:3, 3:4, 1:1, 21:9）와 **수량**（1-9 장）을 중심으로 구축되어 콘텐츠의 플랫폼, 톤, 주제, 독자의 기대에 맞춰 창의적 출력을 조정합니다.

> "일러스트는 설명해서는 안 된다 —— 확장해야 한다."

## 주요 기능

### 듀얼 캐릭터 앵커 모드

브랜드 일관성과 콘텐츠 고유 창의성 사이에서 선택:

| 측면 | 브랜드 앵커 모드 | 컨텍스트 앵커 모드 |
|------|----------------|------------------|
| **핵심 로직** | 캐릭터 참조 이미지 업로드, 모든 일러스트에 동일한 형상 사용 | 각 글의 주제에 맞는 독창적 캐릭터 생성 |
| **최적 사용처** | 브랜드 콘텐츠, 시리즈 글, 개인 IP, 기업 블로그 | 심층 글, 창의적 실험, 트렌드 주제 |
| **캐릭터 일관성** | 최대 (모든 콘텐츠 동일 형상) | 높음 (한 글 내 동일 형상) |
| **브랜드 인지도** | 강함 (시각적 시그니처) | 미묘함 (콘텐츠 우선 접근) |
| **창작 자유도** | 기존 캐릭터에 제약 | 무제한, 창의성 최대화 |

### 사용자 정의 가능한 화면비와 수량

생성 전에 Skill은 두 가지 간단한 설정 단계를 안내합니다:

**화면비 선택** —— 타겟 플랫폼에 맞는 캔버스 형태를 선택:

| 비율 | 방향 | 최적 사용처 |
|:---:|:---:|:---|
| **16:9** | 가로 | 네이버 블로그, 티스토리, 유튜브 썸네일, 브런치 (기본값) |
| **9:16** | 세로 | 유튜브 쇼츠, 인스타그램 릴스, 틱톡, 카카오스토리, 휴 대폰 배경화면 |
| **4:3** | 가로 | 파워포인트, 강의 슬라이드, 보고서, 프레젠테이션 |
| **3:4** | 세로 | 인스타그램 피드, 네이버 포스트, 카카오톡 프로필, 스레드 |
| **1:1** | 정사각형 | 인스타그램, X/트위터, 카카오톡 오픈채팅, 당근마켓, 제품 사진 |
| **21:9** | 초광각 | 웹사이트 배너, X/트위터 헤더, 트위치, 아프리카TV, 와이드스크린 |

Skill은 콘텐츠 유형에 기반하여 지능적으로 추천하지만, 임의의 비율로 덮어쓸 수 있습니다.

**수량 선택** —— 생성할 일러스트 수를 선택（1-9 장）:

| 글 길이 | 추천 수량 |
|:---|:---:|
| 짧은 글（< 1000 자） | 1-3 장 |
| 중간 길이（1000-3000 자） | 4-6 장 |
| 긴 글（> 3000 자） | 7-9 장 |

### 컨셉트 포지: 4가지 창작 방법론

**1. 직역법** —— 글의 핵심 개념을 직접 시각화
**2. 이질적 합성** —— 3개의 무관련 영역을 강제로 조합
**3. 인지적 변위** —— 규모·기능·시공간·감정의 4가지 변위
**4. 랜덤 트리거** —— 소재 + 실체 + 직업 + 감정의 조합

### 사용자 확인 플로우（컨텍스트 앵커 모드）

Skill은 사용자의 승인 없이 캐릭터를 생성하지 않습니다：
1. 글 주제에 기반하여 2-3개의 캐릭터 컨셉트를 제시
2. 사용자의 선택 또는 수정 제안을 대기
3. 승인된 캐릭터 설명을 잠금
4. 글의 모든 일러스트에 동일 캐릭터를 사용

### 다국어 라벨 지원

Skill은 글의 언어를 자동 감지하여 동일한 언어로 라벨을 생성합니다. 수동 선택도 가능：

| 감지 언어 | 라벨 스타일 | 예시 |
|:---|:---|:---|
| **中文** | 가로, 2-8자, 손글씨 | 光芯片 / 数据流 / 瓶颈 |
| **English** | 가로, 1-3단어, 손글씨 | Light Chip / Data Flow / Bottleneck |
| **日本語** | 가로 또는 세로, 짧은 구문 | 光チップ / データ流 / ボトルネック |
| **韓国語** | 가로, 짧은 구문 | 광칩 / 데이터 흐름 / 병목 |

**자동 감지**：글 텍스트 분석（>50% 문자 유형 판단）.**수동 덮어쓰기**：「한국어 라벨로」.

### 퀵 모드 vs 전문가 모드

시간과 제어 필요에 따라 워크플로우 선택：

| 측면 | 퀵 모드 | 전문가 모드 |
|------|--------|------------|
| **단계** | 1단계 자동 감지 | 완전한 9단계 |
| **최적** | 빠른 출력, 숙련 사용자 | 신규 사용자, 정밀 제어 |
| **파라미터** | 모두 자동 감지 | 사용자가 순차 선택 |
| **전략 보고서** | 건너뛰기 | 생성 전 옵션 출력 |

**퀵 모드 예시**：「이 글 일러스트 생성」→ 한국어 자동 감지, 16:9, 6장, 베이스 스케치.

**전문가 모드 예시**：「전문가 모드」→ 모드 → 언어 → 화면비 → 수량 → 스타일 →（옵션 전략 보고서）→ 내용 분석 → 생성.

### 하이브리드 스타일 믹싱（전문가 모드）

두 가지 스타일을 혼합하여 독특한 시각 효과를 만듦：

| 주 스타일（60-70%） | 보조 스타일（30-40%） | 결과 |
|:---|:---|:---|
| 잉크 워시 선 | 사이버 네온 | 「네온 수묵」— 전통 붓놀림에 발광 악센트 |
| 클레이 스톱모션 | 글리치 아트 | 「손상된 점토」— 수제 형태에 디지털 아티팩트 |
| 레트로 테크 | 오가닉 웜스 | 「바이오 메카닉 빈티지」— 낡은 기계에 식물이 무성함 |

스타일 선택 시 **[H] 하이브리드 모드** 선택하여 활성화.

### 세션 기록과 재사용

생성 후 파라미터를 자동 저장. 간단한 명령으로 재사용：

| 명령 | 작업 |
|:---|:---|
| 「저번과 동일」 | 마지막 세션 불러오기 |
| 「기본 설정 사용」 | 저장된 기본값 불러오기 |
| 「기본으로 저장」 | 현재 파라미터 저장 |
| 「기록 보기」 | 최근 10건 목록 |
| 「3번 세션 재사용」 | 지정 세션 불러오기 |

기록은 로컬 `assets/session-history.json`에 저장 — 데이터는 외부로 전송되지 않음.

### 시리즈 캐릭터 연속

시리즈 글을 쓰나요? 컨텍스트 앵커 캐릭터를 저장하여 여러 글에서 재사용：

1. **저장**：「AI Weekly 시리즈용 저장」→ 세션 기록에 저장
2. **재사용**：「AI Weekly 캐릭터 사용」→ 확인 건너뛰기, 저장된 캐릭터 사용
3. **관리**：목록, 업데이트, 퇴역, 클론

| 기능 | 브랜드 앵커 | 시리즈 캐릭터 | 컨텍스트 앵커 |
|------|------------|--------------|--------------|
| 범위 | 모든 콘텐츠 | 하나의 시리즈 | 하나의 글 |
| 지속성 | 영구 | 반영구 | 임시 |
| 최적 | 개인 IP | 칼럼, Newsletter | 단발 글 |

### 일러스트 전략 보고서（전문가 모드）

생성 전 전략 보고서를 요청하여 시각적 사고를 이해：

```
「먼저 전략 보여줘」
```

보고서에는 다음이 포함됨：
- **콘텐츠 분석**：글 유형, 핵심 논점, 타겟 독자
- **시각 앵커 포인트**：어떤 섹션에 이미지가 필요한지, 왜인지
- **인지 부하 맵**：텍스트 전용 vs 이미지 지원 섹션
- **캐릭터 전략**：왜 이 캐릭터가 콘텐츠에 적합한지
- **스타일 이유**：왜 이 스타일이 톤에 맞는지
- **수량과 배치**：이미지 수와 배치 로직

이미지 생성을 교육 가능한 콘텐츠 디자인 프로세스로 전환.

### 첫 사용 가이드

첫 사용자는 자동으로 **단계별 가이드**에 들어갑니다：

1. **웰컴**：친근한 소개, 언제든 스킵 가능
2. **모드 설명**：브랜드 앵커 vs 컨텍스트 앵커 비교와 용도
3. **언어 선택**：각 언어 옵션의 명확한 설명
4. **화면비 가이드**：시각적 형태 프리뷰 + 플랫폼 추천
5. **수량 가이드**：글자 수→이미지 수 매핑 + 인지적 목적 설명
6. **스타일 갤러리**：각 스타일에 이모지 아이콘, 설명, 최적 용도
7. **콘텐츠 입력**：지원 형식 설명
8. **전략 보고서**：옵션 생성 전 분석
9. **캐릭터 선택**：가이드付き 캐릭터 컨셉트 비교
10. **완료 요약**：파라미터 요약 + 다음 팁

스킵 명령：「가이드 건너뛰기」/「다시 표시하지 않기」/「퀵 모드」

### 에러 핸들링

일반적인 문제에 명확한 가이던스를 제공：

| 시나리오 | 복구 방법 |
|----------|----------|
| 참조 이미지를 찾을 수 없음 | 업로드 안내 또는 컨텍스트 앵커로 전환 |
| 참조 이미지가 너무 작음/흐림 | 요구사항 설명, 대안 제시 |
| 참조 이미지에 여러 캐릭터 | 어느 것을 사용할지 묻기, 크롭 제안 |
| 글이 너무 짧음（<500자） | 1장의 히어로 이미지 제안 |
| 글이 너무 긺（>5000자） | 섹션별 생성 권장 |
| 캐릭터 불일치 | 키워드 강화, 디자인 단순화 |
| 라벨 언어 오류 | 명시적 선언으로 재생성 |
| 스타일이 올바르게 적용되지 않음 | 스타일 주입 강화로 재생성 |
| 생성 실패（속도 제한） | 30초 대기 후 재시도, 수량 감소 |

모든 에러에：무엇이 잘못되었는지, 왜, 2-3개의 실행 가능한 옵션 포함.
